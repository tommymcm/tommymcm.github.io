Detailed Description

This pass propagates attributes from kernels to the non-entry functions.

Most of the library functions were not compiled for specific ABI, yet will be correctly compiled if proper attrbutes are propagated from the caller.

The pass analyzes call graph and propagates ABI target features through the call graph.

It can run in two modes: as a function or module pass. A function pass simply propagates attributes. A module pass clones functions if there are callers with different ABI. If a function is clonned all call sites will be updated to use a correct clone.

A function pass is limited in functionality but can run early in the pipeline. A module pass is more powerful but has to run late, so misses library folding opportunities.
